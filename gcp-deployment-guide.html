1
Create a VPC Network

The company org policy may have deleted the default VPC. If no default network exists:

  1. GCP Console → VPC networks → Create VPC Network
  2. Name: default
  3. Subnet creation mode: Automatic
  4. Click Create
2
Create a Cloud NAT (for outbound internet)

The org policy blocks external IPs on VMs, so the VM needs NAT for outbound access (pulling Docker images, cloning repos, etc.).

Create a Cloud Router

  1. GCP Console → Cloud Routers → Create Router
  2. Name: default-router
  3. Network: default
  4. Region: us-central1
  5. Click Create

Create a Cloud NAT Gateway

  1. GCP Console → Cloud NAT → Create Cloud NAT gateway
  2. Name: default-nat
  3. Network: default, Region: us-central1
  4. Cloud Router: default-router
  5. Click Create
3
Create the VM
  1. GCP Console → Compute Engine → VM instances → Create Instance
  2. Name: enterprise-rag
  3. Region: us-central1, Zone: us-central1-a
  4. Machine type: e2-medium (2 vCPU, 4 GB RAM)
  5. Boot disk: Click Change → Ubuntu 22.04 LTS, x86/64, 60 GB SSD
  6. Firewall: check Allow HTTP and Allow HTTPS
  7. Network interface: select the default network
  8. Click Create
4
Create a Firewall Rule for Health Checks

The load balancer needs to reach the VM for health checks.

  1. GCP Console → VPC network → Firewall → Create Firewall Rule
  2. Name: allow-health-check
  3. Network: default
  4. Direction: Ingress
  5. Targets: All instances in the network
  6. Source IPv4 ranges: 130.211.0.0/22,35.191.0.0/16
  7. Protocols and ports: TCP, port 80
  8. Click Create
5
SSH into the VM and Install Docker

Click the SSH button next to your VM in the Compute Engine console.

If IAP blocks you, you may need the IAP-secured Tunnel User role added to your account under IAM & Admin → IAM.

Once connected, run:

curl -fsSL https://get.docker.com | sudo sh
sudo apt install -y git
6
Clone the Repository
sudo git clone https://github.com/zaminanwar/Ragmvp2.git /opt/rag
cd /opt/rag
sudo git checkout claude/main-with-gemini
7
Create the Credentials File
sudo mkdir -p /opt/rag/credentials
sudo nano /opt/rag/credentials/service-account.json

Paste the full Vertex AI service account JSON key (with the real private_key).
Save with Ctrl+O, Enter, then exit with Ctrl+X.

8
Create the .env File
sudo cp /opt/rag/.env.example /opt/rag/.env
sudo nano /opt/rag/.env

Update the following settings:

# Change these to random strings
SECRET_KEY=<random-string>
JWT_SECRET_KEY=<random-string>

# Database password (change from default)
POSTGRES_PASSWORD=<strong-password>
DATABASE_URL=postgresql+asyncpg://raguser:<your-password>@postgres:5432/ragdb
DATABASE_SYNC_URL=postgresql://raguser:<your-password>@postgres:5432/ragdb

# MinIO credentials (change from default)
MINIO_ACCESS_KEY=<change-me>
MINIO_SECRET_KEY=<change-me>

# Gemini / Vertex AI
GOOGLE_CLOUD_PROJECT=gcp-000630-042326
GOOGLE_CLOUD_LOCATION=us-central1
GOOGLE_APPLICATION_CREDENTIALS=/app/credentials/service-account.json

# Set Gemini as default LLM
DEFAULT_LLM_PROVIDER=gemini
DEFAULT_LLM_MODEL=gemini-2.5-flash

# Set Gemini as default embedding provider
DEFAULT_EMBEDDING_PROVIDER=gemini
DEFAULT_EMBEDDING_MODEL=text-embedding-004
EMBEDDING_DIMENSIONS=768

Save and exit.

9
Build and Launch
cd /opt/rag
sudo docker compose up -d

This takes 10–15 minutes the first time (pulling images and building containers).

If Elasticsearch fails its health check on the first start (common), just run sudo docker compose up -d again.

10
Run Database Migrations
# Run migrations
sudo docker compose exec backend alembic upgrade head

# Update vector dimension for Gemini embeddings (768 instead of 1536)
sudo docker compose exec postgres psql -U raguser -d ragdb \
  -c "ALTER TABLE document_chunks ALTER COLUMN embedding TYPE vector(768);"
11
Set Up the Load Balancer

Since the org policy blocks external IPs on VMs, a load balancer exposes the app publicly.

Create an Instance Group

  1. GCP Console → Instance groups → Create Instance Group
  2. Select New unmanaged instance group
  3. Name: rag-instance-group
  4. Region: us-central1, Zone: us-central1-a
  5. Network: default
  6. Select your VM from the dropdown
  7. Port mapping: name http, port 80
  8. Click Create

Create the Load Balancer

  1. GCP Console → Load balancing → Create Load Balancer
  2. Type: Application Load Balancer → Public → Global → Global external Application Load Balancer
  3. Name: rag-load-balancer
  4. Frontend configuration: Protocol HTTP, Port 80
  5. Backend configuration: Create a backend service
    • Name: rag-backend-service
    • Backend type: Instance group
    • Select rag-instance-group, port 80
    • Health check: Create new — name rag-health-check, HTTP, port 80, path /api/health
  6. Click Create

Wait 2–3 minutes for provisioning. The frontend IP shown is your app's public URL.

12
Verify

Open http://<load-balancer-ip> in your browser. You should see the login page.

  1. Register an account
  2. Create a workspace (toggle RAG features as desired)
  3. Upload a document
  4. Chat with it

Updating the App

When code changes are pushed to the repository:

cd /opt/rag
sudo git pull origin claude/main-with-gemini
sudo docker compose up -d --build

For frontend-only changes:

sudo docker compose up -d --build frontend

For backend-only changes:

sudo docker compose up -d --build backend

Architecture Overview

Load Balancer (<public-ip>:80)
  |
  v
Nginx (reverse proxy, port 80)
  |-- /api/*  --> Backend (FastAPI, port 8000)
  |-- /ws/*   --> Backend (WebSocket)
  |-- /*      --> Frontend (React, port 3000)

Backend services:
  - PostgreSQL 16 + pgvector  (vector search)
  - Elasticsearch 8.12        (BM25 full-text search)
  - Redis 7                   (cache, job queue)
  - MinIO                     (S3-compatible document storage)
  - Collector                 (async document processor)
  - Ollama                    (optional local LLM)
